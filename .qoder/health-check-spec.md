## 健康检查接口实现规范

### 需求描述
为 CdcHttpServer 服务增加健康检查接口，用于监控服务运行状态。

### 功能要求

#### 场景：健康检查端点
- **当** 服务启动并监听端口
- **则** 应提供 `GET /health` 接口
- **且** 接口返回 HTTP 200 状态码
- **且** 接口返回文本内容 "OK"

#### 场景：现有路由保持
- **当** 访问根路径 `/`
- **则** 返回 "CDC HTTP Server is running"
- **且** 现有 `/cdc` POST 接口功能不变

### 技术约束
- 使用 .NET 并非修改任何现有代码结构
- 仅添加新的健康检查端点
- 保持所有现有路由的处理逻辑不变

### 验证标准
1. 服务启动后能正常监听端口
2. GET /health 返回 200 状态码和 "OK" 文本
3. 所有现有路由保持正常运行
