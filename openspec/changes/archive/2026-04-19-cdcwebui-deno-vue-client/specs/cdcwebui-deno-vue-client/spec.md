## 新增需求

### 需求：事件列表展示
客户端应展示从 `/api/events` 接口获取的 CDC 事件列表。

#### 场景：展示事件
- **当** 客户端加载时
- **则** 应从 `/api/events` 获取事件并在列表中展示

#### 场景：事件信息显示
- **当** 查看列表中的事件时
- **则** 应显示：时间戳、数据库、表名、操作类型（CREATE/UPDATE/DELETE/READ）和数据预览

### 需求：实时更新
客户端应定期自动刷新事件数据。

#### 场景：自动刷新
- **当** 客户端运行时
- **则** 应每 3 秒（可配置）轮询 `/api/events`

#### 场景：新事件视觉提示
- **当** 新事件到达时
- **则** 应以高亮或动画方式提示新数据

### 需求：统计信息展示
客户端应展示来自 `/api/stats` 接口的统计信息。

#### 场景：统计面板
- **当** 客户端加载时
- **则** 应显示：总事件数、最近1分钟事件数、最近1小时事件数

#### 场景：统计自动刷新
- **当** 统计面板可见时
- **则** 应与事件使用相同的间隔自动更新

### 需求：事件筛选
客户端应提供事件列表的筛选功能。

#### 场景：按操作类型筛选
- **当** 用户选择操作类型（CREATE/UPDATE/DELETE/READ）
- **则** 列表应仅显示该类型的事件

#### 场景：按表名筛选
- **当** 用户在搜索框中输入表名
- **则** 列表应仅显示匹配表的事件

#### 场景：清除筛选
- **当** 用户点击清除按钮
- **则** 应重置所有筛选并显示完整列表

### 需求：事件详情查看
客户端应允许查看完整的事件详情。

#### 场景：查看详情
- **当** 用户点击事件
- **则** 详情面板或弹窗应以格式化 JSON 显示完整事件数据

#### 场景：复制事件数据
- **当** 用户在详情视图中点击复制按钮
- **则** 事件 JSON 应被复制到剪贴板

### 需求：响应式设计
客户端应在不同屏幕尺寸下可用。

#### 场景：桌面视图
- **当** 在桌面浏览器（>1024px）中查看时
- **则** 布局应显示侧边栏统计和主事件列表

#### 场景：移动视图
- **当** 在移动设备（<768px）上查看时
- **则** 布局应适配为单列，面板可折叠

### 需求：技术栈
客户端应使用 Deno 和 Vue 3 构建。

#### 场景：Deno 运行时
- **当** 运行开发服务器时
- **则** 应使用 Deno 2.x 运行时

#### 场景：Vue 3 框架
- **当** 开发组件时
- **则** 应使用 Vue 3 组合式 API

#### 场景：TypeScript 支持
- **当** 编写代码时
- **则** 所有文件应使用 TypeScript 并带有正确的类型定义
