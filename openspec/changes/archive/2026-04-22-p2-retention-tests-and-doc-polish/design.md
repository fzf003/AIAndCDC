## Context

当前项目的事件存储仍以 `InMemoryEventStore` 为主，这对演示足够，但对长期演进并不充分。同时，关键链路缺少自动化测试，本地运行文档也还没有收敛成一套可交接的中文材料。

P2 的目标是把这条 CDC 主线补到“可以持续演进”的水平。

## Goals / Non-Goals

**Goals**

- 明确事件存储的当前定位与后续扩展方向。
- 为关键链路建立最小可用测试集。
- 提供一套清晰、中文为主、可复现的本地启动说明。

**Non-Goals**

- 本阶段不强制落地完整持久化实现。
- 本阶段不引入复杂监控平台或端到端部署方案。

## Decisions

### 1. 保留内存存储作为默认实现

`InMemoryEventStore` 继续作为默认演示实现保留。

同时明确扩展接口，使未来可以平滑增加持久化版本，而不破坏现有 API。

### 2. 测试优先覆盖主链路

优先补以下测试：

- Debezium 事件解析
- `/api/events`
- `/api/events/{id}`
- `/api/stats`
- Redis 消费器处理器匹配与 ACK 行为

### 3. 文档只保留一套推荐运行方式

P2 的文档收口建立在 P0/P1 决策基础上，只保留一套推荐启动路径与排障路径。

## Risks / Trade-offs

- **风险：测试补齐会暴露现有结构问题**
  - 缓解：这正是 P2 的价值，应接受并迭代修正。
- **权衡：暂不引入持久化实现，长期能力仍有限**
  - 接受：先把接口与测试边界铺好，再决定是否上 SQLite 或其他存储。
